What I can help with.

Gameplay Systems (UE5 C++ / Blueprints)
I build responsive gameplay mechanics in Unreal Engine 5: movement and camera feel, input-driven actions, interaction systems, HUD/UI hooks, and clean gameplay state management. I’m comfortable mixing C++ for math-heavy logic and performance-critical code with Blueprints for fast iteration, prototyping, and designer-friendly tuning. I work with debug overlays, profiling, and structured logs to diagnose “feel” issues like overshoot, jitter, and control instability.
01
Multiplayer Foundations (Prediction / Replication)
I design gameplay code with networking in mind: authoritative state, input as the source of truth, and predictable simulation. I’m familiar with UE5 workflows around client prediction and reconciliation, testing under artificial latency, and keeping controls stable when roll/pitch/turn rates interact. My focus is building systems that remain readable and maintainable while supporting future multiplayer expansion and performance constraints.
02
Tools & Low-Level Engineering (C99 / SQLite)
I write low-level C (C99) with a focus on correctness, memory discipline, and performance. I’ve built custom tooling around structured parsing, data pipelines, and SQLite-backed storage, including RAG-style retrieval and automated self-correction loops for development workflows. I’m comfortable reading docs, refactoring older habits, and building systems that are testable, reproducible, and easy to debug with clear instrumentation.
03
Prototyping, Content & Production Support
I ship playable prototypes and presentable vertical slices: fast UE5 iteration, VR testing (Quest 2), level setup, lighting/post-process tuning, VFX integration, and production-friendly organization. I can also support content workflows (Maya/Blender basics, texture map prep, normal/roughness/metallic pipelines) and build reliable version-control habits for teams. The goal is always the same: make the game feel good, look readable, and stay stable as it grows.
